# Salesforce Test Automation & DevSecOps Portfolio
**Target Role:** Salesforce Test Engineer (DoD / Federal Compliance Focus)

This portfolio demonstrates hands-on experience building automated testing frameworks, validating Salesforce security models, and creating documentation aligned with DoD Risk Management Framework (RMF) and Authority to Operate (ATO) readiness.

## Featured Projects

### 1. Salesforce UI & LWC Automation Framework
* **Objective:** Validate Lightning Web Components (LWC) and complex Salesforce Flows.
* **Tools:** Playwright, JavaScript, GitHub Actions.
* **Key Achievement:** Automated end-to-end user flows for custom objects, ensuring permission sets and validation rules trigger correctly across different user profiles.

### 2. Flosum-Inspired CI/CD Pipeline Gate Simulation
* **Objective:** Ensure code quality and security health checks run automatically prior to deployment.
* **Tools:** GitHub Workflows, Python.
* **Key Achievement:** Simulates a Flosum pipeline gate that automatically blocks deployments if static analysis flags missing Salesforce Shield encryption or open Apex sharing violations.

### 3. REST API & Integration Testing
* **Objective:** Verify data integrity between Salesforce and external federal middleware.
* **Tools:** Postman / JavaScript.
* **Key Achievement:** Built automation scripts to test RESTful APIs, verifying payload structure, response times, and OAuth 2.0 authentication mechanisms.

### 4. DoD Compliance & Section 508 Accessibility Auditing
* **Objective:** Document testing protocols for ATO readiness and accessibility compliance.
* **Artifacts Included:** * `ATO_Readiness_Artifact.md`: Detailed testing log for Salesforce Shield and Field-Level Encryption (FLE).
    * `Section_508_Accessibility_Log.md`: Section 508 keyboard navigation and screen-reader test matrix for Salesforce Communities/Experience Cloud.
